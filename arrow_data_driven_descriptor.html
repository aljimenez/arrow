Data Driven Descriptor

If you've a use case, where you can re-use the same descriptor with different test data, data driven descriptors come handy. You can put your test data directly in the descriptor or in a separate data file (JSON format).

1) Putting test data directly in the descriptor You can put the test data as an array of config objects in the descriptor as shown below. ""finance" and "yahoo" are the keys for your data items which needs to be unique. 


{
    "config": [
        {
            "finance": {
                "baseUrl": "http://finance.yahoo.com",
                "searchBtnId": "#mnp-search_box"
            }
        },
        {
            "yahoo": {
                "baseUrl": "http://yahoo.com",
                "searchBtnId": "#p_13838465-p"
            }
        }
    ]
}

2) Putting test data in a data file (JSON format) You can declare the location of your data driver file in the descriptor as shown below 
                    "dataDriver" : "./configdata.json",

The configdata.json will look like 
{
    "config": [
        {
            "finance": {
                "baseUrl": "http://finance.yahoo.com",
                "searchBtnId": "#mnp-search_box"
            }
        },
        {
            "yahoo": {
                "baseUrl": "http://yahoo.com",
                "searchBtnId": "#p_13838465-p"
            }
        }
    ]
}

For the above config data, Arrow will iterate over this descriptor 2 times, once for each array element in the "config" object.
